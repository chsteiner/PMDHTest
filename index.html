Was ist CSS?

May 3, 2022 by Viktoria Köhnlein

CSS = Cascading Style Sheets
CSS ist eine Formatierungssprache, welche vorwiegend für HTML (aber auch SVG) entwickelt wurde. Sie erlaubt das Modifizieren einzelner HTML-Elemente.

Funktionen

CSS legt die Schriftart und -größe fest
setzt Farben, Hintergrundfarben und Hintergrundbilder
positioniert Header, Sidebar und Footer für das Layout
und bewegt die Webseite mit einfachen Animationen.

Wodurch zeichnet sich CSS aus?

Flexibilität: Im Vergleich einfaches Überarbeiten des Layouts möglich.
Konkrete Vorteile sind u.a.:
Kontrolle über das Layout vieler Webseiten aus einer einzigen Style-Sheet-Datei heraus
präzisere Kontrolle über das Layou
verschiedene Layouts für verschiedene Medientypen (Bildschirmanzeige, Druck etc.)
eine Vielzahl von fortgeschrittenen und anspruchsvolle Techniken

Neben HTML und JavaScript eine der Kernsprachen des WorldWideWeb

Syntax

z.B.: h1 { font-size:200px}
h1: selector, { font-size:200px }: declaration wobei: "font-size"=property , "200px"= value mehrere deklarationen in klammer möglich, getrennt durch ";"

Einbindung

Einbindung des CSS-Stylesheets auf verschiedene Arten möglich

  • Interne Einbidung: Direkte Festlegung von Formaten ="Inline" über das "style" Element/Attribut; Diese Modifikationen sind an ein Element gebunden; Nachteile: Verringerung der Flexibilität und Erhöhung des Wartungsaufwandes.
  • Externe Einbindung: Erstellung eines externen Stylesheets/ CSS-Datei. Einbinden über "link"-Element im "head" der HTML-Datei. Vorteile: Ermöglicht einheitliche Formatvorgaben, HTML Dokument wird "übersichtlicher", spätere Design-Änderungen können leicht durchgeführt werden

Bei Einbindung beteiligte Elemente und Attribute

  1. style: Inline Formatierung
  2. rel: erklärt logischen Zusammenhang/Beziehung zwischen HTMl und CSS-Datei. ="stylesheet"
  3. href: referenziert einzubindende CSS-Datei
  4. weitere optionale Attribute: "type" (=text/css), "media": gibt an, für was für Ausgabemedium (z.B.: "print") das Dokument optimiert wurde.

Generelle Vorgehensweise

  1. HTML Datei erstellen, ohne CSS
  2. Nach Beenden -mind. eine CSS Datei erstellen
  3. Einbinden beliebig vieler Stylesheets möglich. Reihenfolge beachten! - Stylesheet welches als unterstes eingebunden wurde überschreibt die vorangegangenen.

Pseudoklassen und /-Elemente

  1. Pseudoelemente: bezieht sich auf ein reales Element in dem korrespondierneden (HTML)-Dokument. z.B.: "First-Letter"
  2. Pseudoklassen: Dasselbe Element kann verschiedene "style"-Eigenschaften annehmen, abhängig von externener Information, welche sich nicht im Dokument widerspiegelt. z.B.:link,visited,active
    z.B.: Hyperlink vor und nach Klicken.
  3. Funktion: Erlaubt Modifikation von (dynamischen) und typograhpischen Effekten, welche nicht durch das Mapping von CSS zu HTML Elementen passieren kann.

Was bedeutet HTTP?

HTTP steht für „Hypertext Transfer Protocol“. Es wurde von Tim Berners-Lee am CERN (Schweiz) entwickelt und bildet die Grundlage des World Wide Web. HTTP regelt nun, wie eine Seite vom Server zum Client übertragen wird.1

Was ist der Zweck von HTTP?

HTTP bildet die Kommunikationsprache in der der Webbrowser mit dem Webserver spricht, um ihm mitzuteilen, was von ihm verlangt wird. Es lässt sich als Pendant zu unserem alltäglichen Gebrauch von Sprache darstellen.2

Wie funktoniert nun HTTP?

Bild entnommen aus IONOS Digital Guide 3

Die Funktionsweise lässt sich am besten anhand 4 Punkte erläutern:

  1. Der/Die Nutzer:in tippt eine Adresszeile in seinen/ihren Internetbrowser ein. (Bsp. example.com)
  2. Der Browser sendet als nächstes eine entsprechende Anfrage, den HTTP-Request, an den zuständigen Webserver, der die gewühnschte Domain verwaltet. Die gängige Request hierbei lautet: "Sende mir die Datei zu". Alternativ kann der Client auch einfach fragen "Hast du die Datei".
  3. Der Webserver epmfängt den Request, sucht die gewüschnte Datei (hier wäre es die Startseite von example.com, also das index.html) und sendet zunächst den Header, der dem anfragenden Client durch einen Status-Code das Resultat seiner Suche mitteilt. Anschließend wenn die Datei vom Server gefunden wurde und der Client sie zugesendet haben will, sendet der Server nach dem Header den Message Body. (In unserem Fall wäre es index.html)
  4. Der Browser empfängt jene Datei und stellt diese als Website dar.4

Die verschiedenen HTTP Versionen

  1. Die Urversion: HTTP/1
  2. Der erste ofizielle Standart: HTTP/1.1
  3. Die erste Erneuerung: HTTP/2
  4. Die Zukunft: HTTP/3 5

OSI - Open Systems Interconnection Model

09.05.2022 Chirstopher Pollin

Das ISO/OSI-Referenzmodell (englisch Open Systems Interconnection model) ist ein Referenzmodell für Netzwerkprotokolle als Schichtenarchitektur. Es wird seit 1983 von der International Telecommunication Union (ITU) und seit 1984 auch von der International Organization for Standardization (ISO) als Standard veröffentlicht. Seine Entwicklung begann im Jahr 1977.

  • 7 Anwendungsschicht (application layer): SMTP, FTP, HTTP … Web
  • 6 Darstellungsschicht (presentation layer): Format Data, Encryption, SSL
  • 5 Sitzungsschicht (session layer): Start & Stop Session, API
  • 4 Transportschicht (transport layer): TCP, UDP: Pakete transportieren
  • 3 Vermittlungsschicht (network layer): IP Address: Switching und Routing
  • 2 Sicherungsschicht (data link): MAC Address, Ethernet
  • 1 Bitübertragungsschicht (physical layer): Cable, Network cards, Hubs

Was sind CSS Processors?

May 2, 2022 Ines Lesjak

Dieser Blog stellt CSS-Prozessoren und die Vorteile von CSS-Prozessoren vor. Es zeigt auch einige verschiedene Funktionenalitäten, die bei der Arbeit an komplexeren Projekten mit CSS nützlich sein können.

CSS sorgt dafür, dass eine Webseite gut aussieht und schneller geladen wird. Bei komplexeren Systemen oder größeren Projekten kann CSS jedoch eine Herausforderung für die Verwendung und Verwaltung darstellen, da es an Reusability (Wiederverwendbarkeit) mangelt. CSS-Prozessoren bieten daher erweiterte Funktionen an, die den Code in CSS erweiterbar, wiederverwendbar und wartbar (maintainable) machen können, da sie in ihren CSS-Code Logik einfügen können (CSS-Preprocessors Examples, 2018).

CSS-Prozessoren können Stylesheets gut organisiert, flexibel, wiederverwendbar und wartungsfähiger machen, damit man einen besseren und sauberen CSS-Code schreiben kann.

CSS Preprocessors Overview (Sinha, 2020)

CSS-Preprocessors

Es gibt eine Reihe verschiedener CSS-Preprocessors, aber die beliebtesten sind Sass und LESS. Ihre nützlichsten Eigenschaften sind Variablen (variables), Verschachtelungen (nesting), Mixins, Operationen, Funktionen, if/else-Anweisungen. CSS-Preprocessors kompilieren den Code, der mit einem speziellen Compiler geschrieben wird. Daraus erstellen sie dann eine CSS-Datei, auf die vom Haupt-HTML-Dokument verwiesen werden kann (CSS-Preprocessors, 2020).

(Sinha, 2020)

Warum CSS-Processors nutzen?

CSS-Processors vereinfachen, beschleunigen und klären die CSS-Entwicklung und stellen sicher, dass Entwickler dasselbe CSS nicht zwei oder mehrmal neu schreiben müssen. Auf diese Weise ist der Code sauberer, einige Teile davon sind wiederverwendbar, es gibt auch mehr Flexibilität, Snippets und Libraries können gemeinsam genutzt werden und es ist einfacher, ein browserübergreifendes CSS zu erstellen (CSS-Preprocessors, 2020).

  • Variables
    • $yourcolor: #000056.yourDiv { color: $yourcolor;}
  • Loops
    • @for $vfoo 35px to 85px {.margin-#{vfoo} {margin: $vfoo 10px; }}
  • If/Else Statements
    • @if width(body) > 500px {background color: blue;} else { background color: white;}

    (CSS-Preprocessors, 2020)

Sass (Syntactically Awesome Style Sheet)

SASS ist ein Open-Source-CSS-Processor. Es wurde 2006 veröffentlicht und gilt als der älteste CSS-Prozessor (CSS-Preprocessors, 2022). SASS ermöglicht Frontend-Entwicklern, Variablen, if/else Anweisungen, for/while/each-Loops, Vererbung und andere Rechenlogik in ihrem CSS-Code zu verwenden (ebd.).

Nützliche integrierte Funktionen

  • @import - hilft beim Einfügen von CSS-Snippets in andere Stylesheets
  • @extend - ermöglicht die gemeinsame Nutzung eines Sets von CSS-Properties von einem Selektor zu einem anderen innerhalb desselben Stylesheets
  • @mixin - hilft beim Erstellen von CSS-Code, der auf der gesamten Website wiederverwendet werden kann

(Zimmer, 2016)

URI - URL - URN

April 20, 2022 by Florian

Uniform Resource Identifier (URI) dienen uns als IDs für physische und digitale Ressourcen. Wir nutzen sie beim Aufruf von Websites oder beim Versenden von E-Mails. Doch was genau sind URI, URL und URN?

Uniform Resource Identifier

Ein Uniform Resource Identifier (URI) ist nach RFC eine Abfolge von Zeichen, die eine Ressource identifiziert. Dabei kann es sich um physische Dinge, wie ein Buch, oder digitale Objekte, wie eine Website, handeln.

"A URI can be further classified as a locator, a name, or both. The term "Uniform Resource Locator" (URL) refers to the subset of URIs that, in addition to identifying a resource, provide a means of locating the resource by describing its primary access mechanism (e.g., its network "location")."
RFC3986, 1.1.3

Schema nach dem URIs funktionieren. Siehe RFC 3986.

Uniform Resource Locator

Der Uniform Resource Locator (URL) bezeichnet eine Unterform der URIs. Diese ermöglichen nicht nur die Identifizierung, sondern auch die Lokalisierung einer Ressource. Man kann darüber also den physischen Standort, z.B. über die gängigen Protokolle HTTP/HTTPS oder FTP, aufrufen. HTTP würde vor der Ressource stehen, wenn sie vom Typ Web ist und FTP, wenn es sich um eine Datei handelt.

Uniform Resource Name

Das dritte Kürzel URN steht für Uniform Resource Name. Auch dieser ist eine Unterform des URI, folgt aber dem Schema urn. Ein URN soll als dauerhafter und ortsunabhängiger Bezeichner für eine Ressource dienen. Einer Ressource wird also ein eindeutiger und permanenter Name gegeben. Ressource kann hier alles meinen, was sich eindeutig beschreiben lässt, z.B. ein Buch über die ISBN.

URI und URL

Gerade URI und URL werden oft verwechselt. So kann ein URI ein Name, ein Locator oder beides sein, wohingegen eine URL nur als Locator auftritt. Auch hier ist im Hinterkopf zu behalten, dass URLs zwar eine Unterform der URIs darstellen, ein URI aber keine URL ist. URIs identifizieren und müssen nicht als Link fungieren. URLs eben schon, da der Aufruf zur zugehörigen Ressource führt. Dabei enthält die URL auch noch eine Schemabezeichnung, wie: ftp, http, mailto oder telnet.

Die RFC führt Beispiele für URIs an: 


                  ftp://ftp.is.co.za/rfc/rfc1808.txt
                  http://www.ietf.org/rfc/rfc2396.txt
                  ldap://[2001:db8::7]/c=GB?objectClass?one
                  mailto:John.Doe@example.com
                  news:comp.infosystems.www.servers.unix
                  tel:+1-816-555-1212
                  telnet://192.0.2.16:80/

Was ist ein Backend?

May 3, 2022 by Julia

Der Begriff Backend - auch Back End oder Back-End geschrieben - tritt häufig in Verbindung mit dem Begriff Frontend auf. Die beiden Begriffe bilden ein Gegensatzpaar bei der Unterteilung von komplexeren Softwarestrukturen. Während Frontend die für alle sichtbaren Teile eines digitalen Produktes wie einer Webpage ist, ist Backend sozusagen das Gegenteil. Es ist die verborgene Seite des digitalen Produktes. Diese ist dabei natürlich nur für den User, nicht für den Programmierer verborgen (http://www.softselect.de/business-software-glossar/backend).

Das Backend kann mit "hinteres Ende" (https://www.techbold.at/lexikon-eintrag/backend) oder vielleicht passender mit Unterbau (https://it-service.network/it-lexikon/backend) übersetzt werden, weil es sich um den wichtigen, administrativen Teil des digitalen Produktes handelt: Es bildet also die funktionale Grundlage beispielsweise einer Webpage und deren Inhalte (https://www.techbold.at/lexikon-eintrag/backend).

Das Backend ist für die Datenverarbeitung im Hintergrund zuständig. Hierbei spielen vor allem die Programmierung zum Beispiel mit Python, der Aufbau des Servers, die Datenbankadministration, die Sicherheit, die Datenübertragung und Backup eine zentrale Rolle. (https://kinsta.com/wp-content/uploads/2021/11)

Wissenschaftliche Einsatzbereiche des Backends sind ein relativ neuer Trend. So wertet beispielsweise die noch junge Disziplin der Computational Social Science die großen Datenmengen der Online-Nutzung im Backend-Bereich aus (Fotis Jannidis / Hubertus Kohle / Malte Rehbein (Hg.), Digital Humanities. Eine Einführung, Stuttgart 2017, S. 30).

Was sind CSS Frameworks?

09.05.2022 von Christian Steiner

CSS-Frameworks sind Werkzeuge, die EntwickleInnen von Web-Interfaces die Arbeit erleichtern.

Anstatt das Rad jedes Mal neu zu erfinden, wenn ein neues Projekt ansteht, geben Frameworks den Entwicklern die Werkzeuge an die Hand, mit denen sie schnell Benutzeroberflächen erstellen können, die im Laufe eines Projekts optimiert und wiederholt werden können, anstatt Zeit damit zu verbringen, mit einem leeren Dokument zu beginnen.

In seiner einfachsten Form ist ein CSS-Framework eine Sammlung von CSS-Stylesheets, die bereits vorbereitet und einsatzbereit sind. Sie sind auf gängige Situationen zugeschnitten, wie z. B. die Einrichtung von Navigationsleisten, und werden oft durch andere Technologien wie SASS und JavaScript erweitert.

CSS Frameworks erleichtern es ungemeint Webseiten responsive zu gestalten, also auf alle Bildschirmgrößen anzupassen

Teamwork

Die Zeitersparnis ist nicht der einzige Vorteil, den der Einsatz von CSS-Frameworks mit sich bringt. Teams mit mehreren Entwicklern, vor allem größere, wissen die Standards, die Frameworks mit sich bringen, sehr zu schätzen. Anstatt dass jeder Entwickler seine eigene Art von Klassennamen in ein Projekt einbringt, standardisieren Frameworks die Layouts und ermöglichen es einem Entwickler, den Code eines anderen Entwicklers problemlos zu lesen. Das spart nicht nur Zeit, sondern sorgt auch für einen reibungsloseren Entwicklungszyklus mit weniger Fehlern und einer einfacheren Kommunikation im Team.

Top Frameworks

Derzeit sind folgende fünf CSS Frameworks am beliebtesten:

  • Bootstrap
  • Foundation
  • Bulma
  • Tailwind
  • UIKit

Was ist HTML?

May 2, 2022 by Georg Reiter

HTML ist die Abkürzung für hypertext markup language. HTML ist – wie der Name schon sagt – keine Programmier-, sondern eine Auszeichnungssprache, mit der Webseiten und deren Inhalte strukturiert werden. Mit dem Markup wird Webbrowsern mitgeteilt, wie der Inhalt einer Webseite – z. B. Text und Bilder – dargestellt werden sollen. „Hypertext“ bezeichnet hierbei die Art und Weise, wie Nutzer*innen sich im World Wide Web bewegen: Durch das Klicken auf Hyperlinks werden sie zu anderen Seiten weitergeleitet, „hyper“ meint hierbei, dass dieses Weiterbewegen nicht linear ist, sondern dass sich die Nutzer*innen zu beliebigen Stellen weiterbewegen können.

Was ist ein HTML-Dokument?

Ein HTML-Dokument ist im Wesentlichen ein Textdokument, dass einer bestimmten, leicht zu erlernenden Syntax und Bennenungskonventionen folgt, die Computern und Webservern anzeigen, dass es sich eben um HTML-Dokument handelt und es auch so interpretiert werden soll. Die wichtigste Konvention ist, dass zu Beginn des Textdokuments der Dokumenttyp „HTML“ deklariert wird: <!DOCTYPE html>. Ebenfalls wichtig ist, dass das Textdokument dann mit der Endung .html abgespeichert wird. Somit ist das Dokument von „innen“ und „außen“ als HTML-Dokument erkennbar.

Aufbau von HTML-Dokumenten

Elemente

HTML-Dokumente bestehen in der Regel aus einer Reihe von sogenannten Elementen, mit denen Inhalte umschlossen werden. Ein Element besteht aus 3 Teilen: 1.) Dem öffnenden Tag, 2.) dem Inhalt und 3.) dem schließendenden Tag.

  1. Öffnendes Tag: Das öffnende Tag beginnt mit einer Spitzklammer, dieser folgt der Elementname, danach folgt wieder einer Spitzklammer.
  2. Inhalt: Dieser wird von den öffnenden und schließenden Tags umschlossen.
  3. Schließendes Tag: Wie das öffnende Tag beginnt das schließende Tag ebenfalls mit einer Spitzklammer, danach folgt allerdings ein Slash. Es endet ebenfalls mit einer Spitzklammer.

Elemente können auch verschachtelt werden, ebenfalls gibt es einige wenige leere Elemente wie z. B. <img>. HTML-Elemente können ebenfalls Attribute haben. Die Attribute (beispielsweise „style“, „id“ und „class“) werden im öffnenden Tag festgelegt. Sie haben Werte, die dazu dienen, das Element näher zu spezifizieren. Ebenfalls eröffnen sie die Möglichkeit, es zielgenau über CSS zu stylen oder es mittels JavaScript zu manipulieren.

Elemente werden miteinander kombiniert – so entsteht eine HTML-Seite. Betrachten wir folgendes basales Beispiel:

  • <!DOCTYPE html>: Hiermit wird der Dokumenttyp deklariert, dies ist obligatorisch und auch notwendig, damit sich das Dokument richtig verhält.
  • <html></html>: Das HTML-Element, oft auch Wurzelelement genannt, umschließt den Inhalt der gesamten Webseite.
  • <head></head>: Im Head-Element wird all das abgelegt, das nicht Inhalt der Webseite ist. Darunter fallen z. B. Keywords, die von Suchmaschinen gefunden werden können, die Beschreibung der Webseite oder Links auf Stylesheets.
  • <meta charset="utf-8">: Hiermit wird der Zeichensatz des Dokuments auf UTF-8 gestellt, was empfohlen ist, weil damit die meisten Zeichen der meisten Sprachen dargestellt werden können.
  • <title></title>: Mit dem Title-Element wird der Titel des Dokuments festgelegt, der z. B. im Tab aufscheint.
  • <body></body>: Das Body-Element umschließt all den Inhalt, den Nutzer*innen der Webseite sehen sollen, z. B. Texte, Videos, Audiodateien usw.

HTML, CSS und JavaScript

Wie bereits eingangs erwähnt, legt HTML hauptsächlich die Struktur von Webseiten fest. Damit Webseiten auch noch gut aussehen bzw. interaktiv sind, müssen CSS und JavaScript verwendet werden. Häufig findet sich bei der Definition des Zusammenspiels dieser dreier Elemente der folgende Vergleich: HTML ist das Gerüst eines Hauses, CSS die Fassade und Inneneinrichtung und JavaScript die Strom- und Wasserleitungen, die das Haus erst bewohnbar machen.

Was sind CMS?

09.05.2022 von Christian Steiner

Ein Content-Management-System, oft abgekürzt als CMS, ist eine Software, mit der Benutzer Inhalte auf einer Website erstellen, verwalten und ändern können, ohne dass dafür spezielle technische Kenntnisse erforderlich sind.

Content-Management-System

  • Application mit der Inhalte im Web veröffentlicht und verwaltet werden können
  • Website Building Tools integriert
  • Authentifizierungsmethoden integriert
  • Plugins und Extensions für Zusatzfunktionen
  • Oft aus Blogging Systemen hervorgegangen
  • Ideal für schnelles Erstellen und Sharen von Inhalten

CMS regeln alle Schritte der Websiteerstellung

Was macht ein Content-Management-System aus?

Technisch gesehen besteht ein Content-Management-System aus zwei Hauptbestandteilen:

  • Eine Content-Management-Anwendung (CMA) - das ist der Teil, mit dem Sie Inhalte auf Ihrer Website hinzufügen und verwalten können (wie Sie oben gesehen haben).
  • Eine Content-Delivery-Applikation (CDA) - das ist der Backend-Prozess hinter den Kulissen, der die von Ihnen in die CMA eingegebenen Inhalte aufnimmt, sie ordnungsgemäß speichert und für Ihre Besucher sichtbar macht.

Welche CMS gibt es?

WordPress ist mit Abstand das beliebteste CMS

Es gibt aber eine Unzahl an verfügbaren CMS. Die Universität Graz setzt zum Beispiel auf TYPO3.

Was sind Webframeworks?

09.05.2022 von Christian Steiner

Ein Web-Framework (WF) oder Web Application Framework (WAF) ist ein Software-Framework, das die Entwicklung von Webanwendungen einschließlich Webdiensten, Webressourcen und Web-APIs unterstützen soll. Web-Frameworks bieten Standardmethoden zur Erstellung und Bereitstellung von Webanwendungen im World Wide Web.

Webframeworks

  • Framework = Programmgerüst
  • Grundlage für Softwareentwicklung
  • Standardfunktionen werden nachgenutzt
  • Auch Web-Application-Framework genannt
  • Zum bauen von Webapps

Prinzipien

  • Don't repeat yourself (DRY)
  • Keep it short and simple (KISS): Ein Problem, eine Lösung
  • Convention over Configuration:
  • Je mehr Konfiguration, desto aufwändiger die Handhabung. Konventionen hingegen bietet eine Möglichkeit, die Komplexität zu reduzieren

Vorteile:

  • Geringerer Zeitaufwand für das Projekt
  • Sauberer Quellcode
  • Größere Community (Support)

Nachteile:

  • Grenzen des Frameworks
  • Codeballast
  • Einarbeitungszeit
  • Schnelllebikeit

Was können Web Frameworks?

Alles was immer wieder auf Webseiten gebraucht wird:

  • Authentifizierung
  • Email
  • Formular Validierung
  • Datenbank Connections

Klassifizierung von Webframeworks

  1. Single-Page vs. Multi-Page
  2. Serverzentriert vs. Clientzentriert

Client:

Angular, vue.js, React (eher Single-Page Apps)

Server:

Django, node.js mit Express, Ruby on Rails (eher Multi-Page Apps)

Häufig gemeinsam eingesetzt (z.B. node.js und React)

Single-Page-Apps (SPA) sind auch in Multi-Page Umgebungen üblich
